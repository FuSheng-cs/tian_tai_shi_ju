# 《天台十句》游戏 PV 脚本

## 基础规格

- 成片长度：65-75 秒，主版本 16:9，后续可裁 9:16 竖版。
- 核心卖点：十句话、AI 实时回应、情绪变化、多结局、文字不是选项而是玩家亲自输入。
- 视觉基准：黑白灰阶、雨夜、像素动画线条、天台栏杆、城市灯光、香烟烟雾。不要用彩色赛博风、血腥冲击、现代 UI 包装或过度炫技转场。
- 文案语气：克制、短句、悬疑，不解释玩法细节太满，把压力留给观众。
- 字体建议：标题用高对比宋体或锐利黑体；机制说明用思源黑体/阿里巴巴普惠体；不要用卡通字体。
- 统一文字样式：白色 `#F2F2F2`，辅助灰 `#9EA3AA`，少量紫色强调 `#B36BFF`。文字加 30%-45% 黑色柔阴影，不加厚描边。

## 可用素材分组

- 游戏录屏 A：菜单进入、开场序章 5 张 CG、进入天台游戏界面。
- 游戏录屏 B：玩家输入一句温和的话，艾自然回复，机会数减少。
- 游戏录屏 C：玩家输入一句刺伤她的话，机会数额外减少，情绪 CG 触发。
- 游戏录屏 D：连续 3-4 轮对话快切，展示“同一句话不同语气会改变反馈”。
- 游戏截图：`output/playwright/tiantaishiju_full_run/*_game_ready.png`、`*_turn_*_reply.png`、`*_final.png`。
- 游戏 CG：`legacy_vue/public/assets/images/cg_opening_stair_01_16_9.webp` 到 `cg_opening_stair_05_16_9.webp`，情绪 CG，结局 CG。

## 录屏执行清单

1. 录屏分辨率用 1920x1080，浏览器缩放 100%，隐藏书签栏和系统通知。
2. 录一段完整进游戏流程：菜单点击、五张开场 CG、进入对话界面，时长约 20 秒。
3. 录一段“好感方向”对话，输入：`我不靠近。你先听我说一句，好吗？`
4. 录一段“刺伤方向”对话，输入：`你只是在用这种方式逼别人注意你。`
5. 录一段“沉默/试探方向”对话，输入：`如果你不想说话，我就在这里等。`
6. 录至少一个情绪 CG 触发片段，以及一个结局结算片段。结局画面只取 2-3 秒，不把全部后日谈文字剪完。

## 字幕与画面文字规则

- 每屏最多两行文字，每行 12 字以内。不要把论坛长介绍直接搬进 PV。
- 机制文案要像刀口一样短：`十句话，不是十个选项。`、`每一句都会留下后果。`、`她不是固定台词。`
- 真实录屏上不要贴大面积文字，避免挡住输入框、剩余机会和艾的表情。
- 所有说明文字从左下或右侧进入，保持 8%-12% 画面边距。
- 标题卡可以用 92px 大字；普通镜头控制在 32-54px；角标控制在 20-24px。

## AI 视频运动提示

- 静态 CG 统一加轻微 2.5D：雨层向下、烟雾向右上、城市灯光轻微闪烁、镜头慢推或慢移。
- 不要让人物五官大幅变形，不要生成新角色，不要让艾从栏杆上做夸张动作。
- 坠落相关镜头只表现失重和距离感，不表现撞击、尸体、血迹。
- UI 录屏不要用 AI 重绘，保留真实界面可信度。
